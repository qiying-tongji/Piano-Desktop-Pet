## 技术栈总览

| 层级 | 技术 | 用途 |
|------|------|------|
| 桌面壳 | Electron 36 | 透明无边框窗、IPC、窗口模式切换 |
| 前端框架 | React 19 + TypeScript + Vite 6 | UI 渲染与热更新 |
| 样式 | TailwindCSS 3 | 全局样式与组件样式 |
| 状态 | Zustand 5 | 轻量全局状态（5 个 store） |
| 3D / 粒子 | Three.js + React Three Fiber + Drei | 背景音乐场粒子 |
| 动画 | Framer Motion 12 | 桌宠悬浮/展开动效 |
| 音频 | Tone.js 15 | Salamander 钢琴采样、混响、短语播放 |
| 手势 | MediaPipe Hand Landmarker | 双手 21 点 landmark 检测 |

## 架构与数据流

### 进程模型

```
┌─────────────────────────────────────────┐
│           Electron 主进程                │
│  main.ts → windowManager → ipc/handlers │
└──────────────────┬──────────────────────┘
                   │ contextBridge (preload)
┌──────────────────▼──────────────────────┐
│         渲染进程 (React + Vite)          │
│  App → PetMode / PianoMode              │
│  Zustand stores + features 模块          │
└─────────────────────────────────────────┘
```

### 核心数据流（手势 → 音乐）

```
摄像头
  → useCamera（获取 MediaStream）
  → useMediaPipeHands（RAF 循环检测）
  → GestureAnalyzer.analyze（特征提取 + One Euro 滤波）
  → IntentDetector.detect（意图 FSM）
  → handleIntentEvent（intentResponses.ts）
  → generatePhrase（aiPianoEngine）+ audioEngine.playPianoPhrase
  → musicIntentStore / audioStore 更新
  → PianoKeyboard 高亮 + MusicFieldParticles 变色
```

### 模式状态机

```
pet（桌宠 220×220）──双击──▶ expanding ──650ms──▶ piano（全屏 MUSIC FIELD）
piano ──点击「收起」──▶ pet
```

## 功能点详解

### 1. 桌宠模式（Pet Mode）

**功能描述**  
在屏幕右下角显示 220×220 透明置顶小窗，内含迷你钢琴图标、粒子与光晕动效。悬停时有提示「双击展开」，双击后播放展开动画并切换至全屏钢琴模式。

| 维度 | 详情 |
|------|------|
| **实现文件** | `src/app/modes/PetMode.tsx`、`src/features/pet/components/DesktopPet.tsx`、`MiniPianoIcon.tsx`、`PetGlow.tsx`、`PetParticles.tsx`、`src/features/pet/hooks/usePetAnimation.ts` |
| **核心函数** | `DesktopPet()`、`usePetAnimation()` → `onClick()`、`onHoverStart()` / `onHoverEnd()` |
| **实现技术** | React、Framer Motion（悬浮/缩放/旋转动画）、React Three Fiber（PetParticles 3D 粒子）、CSS `-webkit-app-region: drag`（窗口拖拽） |
| **实现方式** | `DesktopPet` 外层 `.window-drag` 区域可拖动窗口；中心 96×96 透明按钮 `window-no-drag` 接收双击；`onClick` 先调用 IPC `setAppWindowMode('piano')` resize 窗口，650ms 后 `setMode('piano')` 切换 React 视图 |

---

### 2. 钢琴演奏模式（Piano Mode）

**功能描述**  
全屏「MUSIC FIELD」面板：2 个八度（C3–B4）虚拟键盘，支持鼠标点击、电脑键盘映射、音量/混响/延音踏板、三档性能画质；可收起回桌宠；琴键高亮合并键盘按下、左手和弦、右手短语三类状态。

| 维度 | 详情 |
|------|------|
| **实现文件** | `src/app/modes/PianoMode.tsx`、`src/features/piano/components/PianoKeyboard.tsx`、`PianoKey.tsx`、`MusicFieldParticles.tsx`、`PerformanceQualitySelect.tsx`、`src/features/piano/hooks/usePianoPlay.ts`、`useKeyboardInput.ts`、`src/features/piano/constants/keys.ts` |
| **核心函数** | `PianoMode()`、`PianoKeyboard()`、`usePianoPlay()` → `playNote()` / `releaseNote()`、`useKeyboardInput()` 监听 `KEYBOARD_MAP`、`usePianoWindowMode()` 确保全屏 |
| **实现技术** | React、Tone.js（经 AudioEngine）、Zustand（audioStore、musicIntentStore） |
| **实现方式** | `displayActiveNotes` 合并 `activeNotes` + `activeChordNotes` + `phraseHighlightNotes`；标题栏可拖拽；控制栏调节 volume/reverb/sustain；底部键盘区叠加 `GestureIntentOverlay` |

---

### 3. 音频引擎（Audio Engine）

**功能描述**  
中央音频服务：Salamander 钢琴采样 + Reverb + Volume；键盘 noteOn/noteOff；手势短语 `playPianoPhrase`；左手和弦 `playGestureChord`；电影感/亲密感混响脉冲。

| 维度 | 详情 |
|------|------|
| **实现文件** | `src/features/audio/AudioEngine.ts`、`useAudioEngine.ts`、`src/stores/audioStore.ts` |
| **核心函数/类** | 单例 `audioEngine`：`init()`、`noteOn()` / `noteOff()`、`playPianoPhrase()`、`playGestureChord()`、`applyCinematicClimax()`、`applyIntimateFelt()` |
| **实现技术** | Tone.js（`Tone.Sampler`、`Tone.Reverb`、`Tone.Volume`、`Tone.Sequence`） |
| **实现方式** | 采样从 `https://tonejs.github.io/audio/salamander/` 远程加载；`useAudioEngine` Hook 桥接 Zustand 与引擎，useEffect 同步 volume/reverb/sustain；用户首次交互时 `Tone.start()` 解锁 AudioContext |

**遗留 API（已创建但未接入意图系统）**  
`playGesturePad()`、`playGestureArpeggio()`、`startGestureLoop()`、`playGestureLead()`

### 4. 手势识别（Gesture Recognition）

**功能描述**  
摄像头 → MediaPipe 双手 landmark → 特征提取（速度、张开度、伸指数量）→ Canvas 骨架/能量场叠加。

| 维度 | 详情 |
|------|------|
| **实现文件** | `src/features/gesture/components/HandOverlay.tsx`、`hooks/useCamera.ts`、`useMediaPipeHands.ts`、`useGestureAnalyzer.ts`、`lib/handLandmarker.ts`、`lib/analyzer/extractHandFeatures.ts`（`GestureAnalyzer` 类）、`oneEuroFilter.ts`、`countExtendedFingers.ts` |
| **核心函数/类** | `getHandLandmarker()`、`toHandFrame()`、`GestureAnalyzer.analyze()`、`useGestureAnalyzer().analyzeFrame()`、`useMediaPipeHands()` RAF 检测循环 |
| **实现技术** | `@mediapipe/tasks-vision` HandLandmarker（GPU/CPU 回退）、Canvas 2D |
| **实现方式** | 模型本地 `public/models/` + 远程回退；检测频率受 `performanceProfiles.detectIntervalMs` 节流；HUD 更新受 `hudUpdateMs` 节流；低画质档跳过能量场绘制 |

---

### 5. 手势意图检测（Intent Detection）

**功能描述**  
从手部特征识别 5 类意图：右手四向挥动、双手张开/收拢、左手 1–5 指和弦选择/释放。

| 维度 | 详情 |
|------|------|
| **实现文件** | `src/features/gesture/lib/intent/intentDetector.ts`、`config.ts`、`handRole.ts`、`types.ts` |
| **核心函数/类** | `IntentDetector` 类：`detect()`、`detectDualHand()`、`detectExpression()`、`detectLeftChord()`；`normalizeHandSide()`、`isExpressionHand()` / `isWorldHand()` |
| **实现技术** | 纯 TypeScript 有限状态机 + 阈值配置 |
| **实现方式** | 右手 = 旋律意图（挥动）；左手 = 和弦（伸指数量）；双手距离/openness 变化检测 expand/compress；指数量需稳定 180ms（`CHORD_FINGER_STABLE_MS`）才 latch 和弦；各意图有独立 cooldown |

**意图类型一览**

| 意图类型 | 触发条件 | 手别 |
|----------|----------|------|
| `swipe` | 水平/垂直速度超阈值 + 轴比判定方向 | 右手 |
| `expand` | 双手距离增大 + 平均张开度超阈值 | 双手 |
| `compress` | 双手距离减小 + 张开度低于阈值 | 双手 |
| `chord_select` | 左手伸指 1–5 且稳定 180ms | 左手 |
| `chord_release` | 左手离帧或指数量变化 | 左手 |

---

### 6. 音乐意图引擎 / AI Piano Phrase

**功能描述**  
意图事件 → 生成 0.5–1.5s 钢琴短语 → 播放 + 琴键高亮 + 更新 energy/tension/ambience/pianoMood。

| 维度 | 详情 |
|------|------|
| **实现文件** | `src/features/music-intent/hooks/useMusicIntentEngine.ts`、`lib/intentResponses.ts`、`lib/aiPiano/aiPianoEngine.ts`、`melodicGravity.ts`、`phraseRhythm.ts`、`lib/leftHandChords.ts`、`lib/scales.ts`、`lib/chords.ts` |
| **核心函数** | `useMusicIntentEngine().processSnapshot()` → `IntentDetector.detect()` → `handleIntentEvent()` → `generatePhrase()` / `behaviorFromSwipe()`；`buildGravityPool()`、`phraseRhythm()`、`getLeftHandChordByFingerCount()` |
| **实现技术** | 规则型程序化音乐（非 LLM）：音阶引力、句法记忆、随机节奏 |
| **实现方式** | 管道：`HandOverlay.handleFrame` → `analyzeFrame` → `processSnapshot` → `handleIntentEvent` → `audioEngine.playPianoPhrase` + `musicIntentStore` 状态更新 |

**意图 → 音乐映射（当前生效）**

| 意图 | PhraseBehavior | 音乐效果 |
|------|----------------|----------|
| swipe → | `ascend` | 上行短语 |
| swipe ← | `descend` | 回落收句 |
| swipe ↑ | `lift` | 高音 accent |
| swipe ↓ | `settle` | 稀疏低音 |
| expand | `climax` | 高潮短语 + 混响 swell（`applyCinematicClimax`） |
| compress | `intimate` | 亲密短语 + 干燥声场（`applyIntimateFelt`） |
| chord_select 1–5 指 | — | C / Dm / Em / F / G 三和弦 |
| chord_release | — | 释放和弦 |

**左手和弦映射**

| 伸指数量 | 和弦 |
|----------|------|
| 1 | C |
| 2 | Dm |
| 3 | Em |
| 4 | F |
| 5 | G |

## 项目不足

### 架构与代码健康

1. **遗留/死代码较多**
   - `melodyPicker.ts`、`loopNotes.ts`：旧「单音/琶音/循环」逻辑，当前意图系统未调用
   - `AudioEngine` 中 pad/arpeggio/loop API 已创建但无调用方
   - `musicIntentStore` 的 `loopActive`、`MusicMode`（dream/pulse/drift/ritual）仅有默认值，无 UI 或逻辑驱动
   - `useDraggableWindow.ts` 已实现但未被任何组件引用（当前用 CSS drag）

2. **「AI 桌宠」名不副实**  
   README 称「AI 桌宠」，实际音乐生成是**规则 + 随机**的 procedural phrase，无 LLM/机器学习推理。

### 功能局限

3. **音阶/调式不可配置**  
   `scale` 固定为 `C-pentatonic`，`mode` 固定为 `dream`，UI 无切换入口。

4. **左手和弦映射固定**  
   仅 5 个三和弦（C/Dm/Em/F/G），无转位、七和弦、自定义映射。

5. **键盘音域有限**  
   仅 2 个八度 C3–B4，无滚轮/分页扩展。

6. **远程音频依赖**  
   Salamander 采样从 Tone.js CDN 加载，离线或网络差时初始化失败。

7. **手势体验约束**
   - 必须摄像头 + 良好光照
   - 左右手角色依赖 MediaPipe 的 Left/Right 标签（镜像后可能与用户直觉不一致）
   - 挥动阈值与 cooldown 固定，无校准 UI

8. **平台与交互**
   - 主开发/打包面向 **Windows**；macOS/Linux 未重点验证
   - `setIgnoreMouseEvents` IPC 已预留但渲染层未用，桌宠透明区域无法点击穿透到桌面
   - `usePetAnimation.onClick` 对 `setAppWindowMode('piano')` 调用两次，存在冗余

9. **缺少常见音乐软件能力**  
   无录音、MIDI 输入/输出、预设保存、乐谱、练习模式、多轨等。

## 推荐改进与拓展方向

### 短期（清理 + 体验）

| 方向 | 建议 |
|------|------|
| 死代码治理 | 删除或重新接入 `melodyPicker`/`loopNotes`/pad/arpeggio/loop API |
| 设置面板 | 暴露 scale 切换、左手和弦表编辑、挥动灵敏度、摄像头选择 |
| 本地采样 | 将 Salamander mp3 打入 `public/` 或 app resources，消除 CDN 依赖 |
| 点击穿透 | 在 pet 模式非交互区域调用 `setIgnoreMouseEvents(true, { forward: true })` |
| 窗口拖拽 | 统一 `-webkit-app-region` 与 `useDraggableWindow` 策略 |

### 中期（功能扩展）

| 方向 | 建议 |
|------|------|
| Phrase 引擎增强 | Markov/风格模板/轻量 ML；或 LLM 生成 phrase 参数 |
| MusicMode 落地 | 让 dream/pulse/drift/ritual 影响 `phraseRhythm`、音阶、混响 preset |
| Loop/Hold 手势 | 复用 `startGestureLoop()` + `pickLoopNotes()`，例如「握拳保持」触发 arpeggio loop |
| MIDI | Web MIDI API → `AudioEngine.noteOn`；导出 phrase 为 MIDI |
| 录音 | Tone.Recorder 或 MediaRecorder 导出 WAV |
| 多八度键盘 | 虚拟滚轮或 octave shift 键 |

### 长期（产品化）

| 方向 | 建议 |
|------|------|
| 跨平台 | macOS 透明窗、摄像头权限、electron-builder 多 target |
| 桌宠 AI 人格 | 结合 LLM 做对话气泡、根据演奏/手势给反馈 |
| 练习/教学 | 和弦进行练习、跟弹模式、ConductGuideCard 交互式教程 |
| 性能 | Worker 中跑 MediaPipe；低画质完全跳过 R3F Canvas |
| 预设与云同步 | 用户 chord map、phrase 风格、窗口位置云端保存 |
